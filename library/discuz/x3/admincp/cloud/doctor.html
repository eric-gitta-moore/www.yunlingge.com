11.2. 诊断工具¶

诊断工具

Discuz!云平台诊断工具是帮助您分析站点上的状况，是否能与云平台正常通信等功能。

1、系统状态

站点 ID 是您的站点在云平台的唯一标识，请勿和其他站点共用一套站点 ID 和站点通信 KEY。

2、检测结果

从站点服务器访问QQ互联主接口失败，点击“测试QQ互联其它接口IP”测试其他接口IP是否可以解析成功：

1）如果有任意一个接口解析成功，那么点击“QQ互联域名解析”后的设置接口IP，在弹层里设置QQ互联接口IP为可以解析成功的接口IP。

如果有多个接口IP都解析成功，那么可以填写解析成功耗时最短的那个IP。

2）如果其他接口全部解析失败的话，那么咨询空间商是否开启了防火墙，检查php.ini中是否打开了allow_url_fopen。 如果开启了防火墙，须关闭防火墙或者开放DNS解析，开放服务器发数据到 api.discuz.qq.com 和api.manyou.com。 如果php.ini中allow_url_fopen为off，须改为on。

漫游域名、云平台域名解析失败的解决方法同上。

3、系统插件检测

可能出现的问题：

系统插件状态为系统插件未初始化 (左侧菜单不显示)

原因：这种情况为插件没有正确初始化，具体的表现为后台=>云平台下左侧没有开通的服务的导航。

解决方法：点击系统插件状态后面的点击修复连接进行修复即可。
